---

# 🚀 Why This Project Matters

In modern cybersecurity, **identity is the new security perimeter**.

Organizations rely on access control systems to ensure that users can only access the resources they are authorized to use.

Without proper access control, attackers can escalate privileges and move laterally across systems.

This project demonstrates how **RBAC helps enforce security boundaries within an application environment.**

# 👥 User Roles Simulated

| Role                 | Access Level                    |
| -------------------- | ------------------------------- |
| **Admin**            | Full system access              |
| **Security Analyst** | Access to alerts and monitoring |
| **Employee**         | Limited internal resources      |
| **Guest**            | Restricted view-only access     |

Each role is granted different permissions to demonstrate **authorization controls in action**.

# ⚙️ How It Works

1️⃣ User enters login credentials.
2️⃣ Application validates credentials using a local JSON user database.
3️⃣ The user's assigned role is identified.
4️⃣ Role-based access permissions are applied to the dashboard.
5️⃣ Restricted panels become inaccessible depending on the user's role.

This simulates how enterprise applications enforce **authorization policies**.
